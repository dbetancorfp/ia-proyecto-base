# /develop-backend

Implements the backend plan generated by `/plan-backend-ticket`. Follows TDD: tests first, then implementation.

## Usage

```
/develop-backend ai-specs/changes/GH-42_backend.md
```

## What it does

1. Reads the plan file
2. Creates branch `feature/GH-42-backend`
3. Executes each step in the plan in order
4. Writes failing tests **before** writing implementation code
5. Implements the minimum code to make tests pass
6. Runs `npm test` to verify
7. Runs `npm run typecheck` and `npm run lint`
8. Stages only the files related to the issue
9. Creates a commit: `GH-42: Add user creation endpoint`
10. Pushes and opens a PR with `Closes #42` in the body

## Man-in-the-loop checkpoints

Claude will pause and ask for approval before:
- Writing the first test (shows the test plan)
- Each implementation step (shows what it plans to write)
- The final commit (shows the diff)

You can reject any step and redirect Claude before it proceeds.
